<objective>
Default the `occurredAt` date field to **today (local)** on both capture paths, for entry types
that have an `occurredAt` core field — a default, not a lock. North-star quick win (DATE-01):
one fewer field on the common path.

The default lives in TWO independent places by design (do NOT centralize it in `draftToEntry`):
- **ReviewPage** defaults via its own form-state init, so the user SEES today and can clear it.
- **useShortcutCapture** (one-tap direct-save + sheet-save) defaults the built draft before
  `draftToEntry`.

`draftToEntry` MUST stay neutral: it is shared by ReviewPage.handleSave, where defaulting would
override a user who deliberately cleared the date (violating "default not lock").
</objective>
