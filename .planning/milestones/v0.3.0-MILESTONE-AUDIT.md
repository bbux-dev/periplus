## Integration Check (cross-phase wiring + E2E flows)

Spawned gsd-integration-checker. Result: 13/16 wired on first pass; **1 BLOCKER** + 2 warnings.

### BLOCKER (FIXED in commit 5c99194)
**PORT-01 ↔ PORT-02 round-trip broken.** `buildConfigExportJson` emits an envelope
`{version, exportedAt, config}`, but `importConfig` passed the whole envelope to `migrateConfig`
(which expects `{version, layouts}`), so every exported file was rejected on import with
`'"layouts" must be an array.'`. Unit tests missed it because export and import were tested
independently. **Fix:** `importConfig` now unwraps the envelope's `.config` before validating
(bare hand-edited configs still pass through). Added a round-trip regression test
(`configPort.test.ts` Test 7). Round-trip now verified end-to-end.

### Warnings
- **DASH-02 (FIXED in 5c99194):** renaming the active layout in ManageShortcutsPage did not update
  `activeLayoutRepository`, so the dashboard silently fell back to `layouts[0]` on next visit.
  Now `handleRenameLayout` follows the rename when the renamed layout is the persisted active one.
- **EDIT-03 (ACCEPTED, non-blocking):** the "+ New" chip routes to `/manage` (Add Shortcut one tap
  away) rather than directly to the form. The primary EDIT-03 path — omnibar "Save as Shortcut" —
  navigates directly to the prefilled form and is fully wired. Logged as tech debt.

### E2E Flows — all WIRED
1. Fresh install → seeded dashboard (Phase 12 seeding via Phase 11 repo) ✓
2. Tap shortcut → capture (direct save + undo / hole sheet / confirm→Review) ✓
3. Author shortcut/layout → reflects on dashboard reactively ✓
4. Export → Import round-trip ✓ (after BLOCKER fix)
5. Shared `ShortcutConfig` contract — single key, single validator, no divergent shapes ✓
