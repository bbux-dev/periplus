## Cross-Phase Integration — 4/4 flows PASS

Verified against live source by the integration checker:

1. **Capture flow** — Dashboard → domain → type → CaptureUrlPage → Import (extractMetadataFromUrl) → ReviewPage → Save (entriesRepository.create) → Dexie. PASS
2. **Manual flow** — "Enter Manually" → ManualEntryPage (ENTRY_FIELDS) → buildReviewDraft → same ReviewPage → same Save path (metadata preserved). PASS
3. **Browse/inspect flow** — Dashboard → "View All Entries" → EntryListPage (reactive, domain filter) → row → EntryDetailPage (full entry + metadata JSON + safe sourceUrl) → Back. PASS
4. **Export flow** — EntryListPage → Export → buildExportJson(ALL entries) → triggerDownload. PASS

**Seams confirmed:** the `EntryDomain`/`EntryType` taxonomy in `db.ts` is the single source of truth for both `navigation.ts` and `entryFields.ts`; one `entriesRepository.create` Save path serves both capture and manual; the one `ReviewPage` serves both flows (`ExtractedDraft` structurally compatible with `ReviewDraft`); `isSafeUrl` is shared from `urlUtils.ts` between ReviewPage and EntryDetailPage. No dangling routes; catch-all 404 present.
