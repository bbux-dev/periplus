<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| route param → component | `:domain`, `:type`, and `:id` are untrusted strings supplied via the URL/route. EntryTypePage and the PlaceholderPage stubs read them via useParams. Route matching is handled by react-router; params are a fixed taxonomy in normal use but arbitrary in crafted URLs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service (crash) | EntryTypePage `:domain`/`:type` lookup | mitigate | `getDomainConfig(domain)?.types.find(...)` is fully optional-chained; an unknown domain/type falls back to rendering the raw `type` string in the heading, never dereferencing undefined. Covered by App.test reachability over arbitrary paths. |
| T-03-05 | Tampering / Injection | Stub pages rendering param/title strings | mitigate | Params and the `title` prop are rendered as React text content only (auto-escaped); never HTML, eval, or DB input. No injection surface. |
| T-03-06 | Spoofing (deep-link 404 in PWA) | navigateFallback for parameterized routes | accept | RESEARCH Pitfall 3: existing `navigateFallback: '/index.html'` correctly serves the SPA shell for cold navigations; in-app `<Link>` clicks never hit the SW. No change needed; production-only, low impact. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages in Phase 3 (RESEARCH § Package Legitimacy Audit). No install task in this plan. |
</threat_model>
