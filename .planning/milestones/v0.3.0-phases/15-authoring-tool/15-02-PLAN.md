<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user form input (layout/shortcut names) → config | Untrusted strings enter the config via mutation helpers |
| config object → Dexie persistence | Every write crosses into IndexedDB via configRepository.put |
| config strings → React render | Layout/shortcut names rendered in the manage UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-02 | Tampering | ManageShortcutsPage write handlers | mitigate | Every handler runs validateShortcutConfig(next) and aborts on !ok before configRepository.put; helper throws (duplicate/last-layout) are caught and surfaced |
| T-15-04 | Information Disclosure / XSS | shortcut + layout name rendering | mitigate | All config strings rendered as React text nodes; no dangerouslySetInnerHTML anywhere |
| T-15-05 | Tampering | stale config closure | mitigate | Handlers read FRESH via await configRepository.get() rather than the useShortcutConfig hook value (Pitfall 2) |
</threat_model>
