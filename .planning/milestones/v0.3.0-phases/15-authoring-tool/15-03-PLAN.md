<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user form input (name, dslTemplate, icon, confirm) → config | Untrusted strings enter the config via add/updateShortcut |
| omnibar DSL text → navigation state → form | DSL line carried via location.state into ShortcutFormPage |
| icon selection → config.icon | Icon key persisted to config |
| config strings → React render | Names/templates rendered in form, picker, dashboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | ShortcutFormPage save gate | mitigate | dslTemplate validated by validateTemplate (parseDSL only, no eval); Save disabled and inline error shown when invalid; validateShortcutConfig re-run before configRepository.put |
| T-15-03 | Tampering / Elevation | IconPicker icon selection | mitigate | IconPicker offers ONLY Object.keys(SHORTCUT_ICON_MAP); resolveShortcutIcon falls back to BoltIcon for any unknown key — no arbitrary icon path |
| T-15-04 | Information Disclosure / XSS | form + dashboard string rendering | mitigate | All config strings rendered as React text nodes; no dangerouslySetInnerHTML |
| T-15-05 | Tampering | stale config closure on save | mitigate | handleSave reads FRESH via await configRepository.get() (Pitfall 2) |
</threat_model>
