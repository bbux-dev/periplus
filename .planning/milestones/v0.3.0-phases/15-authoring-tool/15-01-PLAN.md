## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-authored DSL string → parser | Untrusted free-text template enters parseDSL via validateTemplate |
| pure helper output → config persistence | Helper result is later handed to configRepository.put by downstream pages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | validateTemplate / parseDSL | mitigate | Templates are validated via parseDSL ONLY (no eval, no Function ctor); {} token is treated as a plain string and never executed |
| T-15-02 | Tampering | shortcutMutations output | mitigate | Helpers throw on invalid preconditions and return structurally well-formed configs; downstream callers (15-02/15-03) re-run validateShortcutConfig before any put |
| T-15-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies this phase (RESEARCH Package Legitimacy Audit: none) — no install surface |
