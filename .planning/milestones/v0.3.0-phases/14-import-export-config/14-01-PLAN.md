## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | importConfig parse path | mitigate | `JSON.parse` ONLY (no eval/Function); malformed input caught in try/catch and rejected with a reason — no execution of file contents |
| T-14-02 | Tampering | importConfig apply path | mitigate | Run `migrateConfig` (structural validation) before any write; on `{ ok:false }` apply NOTHING (wholesale reject) — `configRepository.put` reached only on `{ ok:true }` |
| T-14-03 | Elevation of Privilege | imported dslTemplate / sourceUrl | accept | dslTemplate stored as inert data; it runs only through `parseDSL` later (Phase 13, no eval) and any produced sourceUrl still passes `isSafeUrl` at save — no new injection surface added here |
| T-14-04 | Denial of Service | newer-version config | mitigate | `migrateConfig` rejects `version > CURRENT_CONFIG_VERSION` wholesale rather than attempting a partial/unknown-shape apply |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | NO new dependencies introduced in this plan — nothing to vet |
</threat_model>
