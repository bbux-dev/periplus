## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Tampering | SettingsPage file handler | mitigate | UI passes the raw File straight to `importConfig`; it applies NOTHING on `{ ok:false }` and only renders the human-readable reason — UI performs no parsing/eval of its own |
| T-14-06 | Information Disclosure | import error message | accept | Error reason originates from the validator (structural, no user-data echo beyond the offending field path) — safe to display |
| T-14-07 | Repudiation/Integrity | error display | mitigate | Error rendered with role="alert" via React-escaped text node (no dangerouslySetInnerHTML) — no markup injection from reason string |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | NO new dependencies — reuses @heroicons/react, react-router-dom, RTL/userEvent already present |
</threat_model>
