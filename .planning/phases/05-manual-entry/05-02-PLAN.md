<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user form input → ManualEntryPage → ReviewDraft → ReviewPage → IndexedDB | Untrusted user-typed values cross into persisted records via location.state |
| route params (:domain/:type) → page logic | Attacker-controllable URL segments select config and persisted domain/type |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | ManualEntryPage :domain/:type params | mitigate | Reuse CaptureUrlPage unknown-domain + unknown-type guards — invalid params render a graceful error and no form; only valid EntryType/EntryDomain reach persistence. ReviewPage re-applies the same guards before save. |
| T-05-02 | Tampering | numeric amount/rating entry | mitigate | buildReviewDraft + ReviewPage parse with parseFloat and skip NaN — no NaN amount persisted. Integration test asserts concrete numeric amounts (45, 120.5). |
| T-05-01 | Tampering/EoP | user-entered title/notes rendered on Review | mitigate | React DOM auto-escapes; values flow as input `value`/JSX text, never `dangerouslySetInnerHTML`. No mitigation change. |
| T-05-03 | Spoofing | sourceUrl | accept | ManualEntryPage renders NO sourceUrl field; manual ReviewDraft.sourceUrl is undefined → ReviewPage skips it. ReviewPage isSafeUrl() guard (Plan 01) remains for the URL-capture path. No added risk. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages in Phase 5 (RESEARCH.md Package Legitimacy Audit: Not applicable). |
</threat_model>
