The new config types this plan defines (the contract Tasks 2 and 3 consume):
```typescript
export interface Shortcut { name: string; icon?: string; dslTemplate: string; confirm: boolean }
export interface Layout   { name: string; icon?: string; shortcuts: Shortcut[] }

export type ValidationResult =
  | { ok: true;  config: ShortcutConfig }
  | { ok: false; reason: string }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported/hand-edited config JSON → app | Untrusted config data (file picker in Phase 14, or hand-edited) crosses into the app and is parsed/validated here before any storage write. |
| stored config → downstream consumers | Config strings (`dslTemplate`, `icon`, `name`) flow to Phases 12–13 for rendering and (later) `parseDSL`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | imported config object → `migrateConfig`/`validateShortcutConfig` | mitigate | Config is data only and is NEVER eval'd. Structural validator rejects malformed config wholesale (no partial apply) before any `configRepository.put`. `dslTemplate` stored inert; only ever parsed by `parseDSL` (no eval) downstream. (CFG-02) |
| T-11-02 | Tampering | crafted `icon` string | mitigate | Icon resolution is a STATIC `SHORTCUT_ICON_MAP` lookup; unknown keys fall back to `BoltIcon`. No dynamic `import()` path exists, so a crafted icon string cannot load an arbitrary module. |
| T-11-03 | Tampering / Spoofing | crafted `name`/`dslTemplate` strings rendered later | accept (Phase 11 surface) | Strings stored as inert data; no rendering in this phase. Downstream rendering uses React JSX string interpolation (no `dangerouslySetInnerHTML`); any produced `sourceUrl` passes `isSafeUrl` at save (Phase 13). No new injection surface introduced here. |
| T-11-04 | Denial of Service | malformed / oversized config JSON | mitigate | `JSON.parse` (caller, Phase 14) throws on malformed JSON; `migrateConfig` returns `{ ok: false }` on structural failure and on non-integer/newer `version`. The error branch must be handled before any Dexie write — no crash, no partial write. |
| T-11-05 | Tampering | future-version config loaded into an older app | mitigate | `migrateConfig` rejects `version > CURRENT_CONFIG_VERSION` with a clear "please update the app" message — prevents loading an unknown future shape. (CFG-03) |
| T-11-SC | Tampering | npm/pnpm installs | N/A — no installs | This phase installs ZERO new runtime or dev packages (RESEARCH Package Legitimacy Audit: N/A). All libraries used are already in `package.json` at verified versions. No install task, no legitimacy checkpoint required. |

No `high`-severity unmitigated threats. V5 Input Validation is the only applicable ASVS category and is satisfied by the whole-or-reject hand-rolled validator.
</threat_model>
