| App code → IndexedDB | LifeLogEntry fields (incl. freeform `title`, `description`, `sourceUrl`, `metadata`) are written to same-origin IndexedDB. All data is local, single-user; no network egress. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-DATA-01 | Tampering | Validation L1 — LifeLogEntry input | accept | Fields are TypeScript-typed at the repository boundary; `title`/`description` are freeform but NOT rendered as HTML in Phase 2 (no JSX dangerouslySetInnerHTML), so there is no XSS sink yet. Re-evaluate when render surfaces land (Phase 6). |
| T-02-DATA-02 | Information Disclosure | IndexedDB storage | accept | Same-origin local store, single-user prototype; no auth, no PII boundary, no sync. Future sync (out of scope) must add transport protection. |
| T-02-DATA-03 | Spoofing | Entry `id` collision | mitigate | Use `crypto.randomUUID()` (122-bit random) not auto-increment, so IDs are globally unique and the future-sync seam cannot collide across devices. |
</threat_model>
