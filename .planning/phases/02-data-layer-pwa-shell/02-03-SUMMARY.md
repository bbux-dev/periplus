## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed invalid tsconfig.node.json glob syntax and test-file scoping issue**
- **Found during:** Task 3 (wire VitePWA into build)
- **Issue:** Plan specified `"src/pwa/**"` in tsconfig.node.json include — TypeScript TS5010 error: "File specification cannot end in a recursive directory wildcard ('**')". Additionally, `"src/pwa/**/*"` (the corrected glob) would include `pwaConfig.test.ts` in the node compilation scope, causing TS2339/TS7006 errors on the `false | Partial<ManifestOptions>` union type
- **Fix:** Used `"src/pwa/pwaConfig.ts"` (specific file) in tsconfig.node.json include. This is functionally equivalent (allows vite.config.ts to import from ./src/pwa/pwaConfig) with no side effects. Also updated the test to use `if (!manifest) return` type-guard to properly narrow `false | Partial<ManifestOptions>` union (required by tsconfig.app.json strict compilation)
- **Files modified:** `tsconfig.node.json`, `src/pwa/pwaConfig.test.ts`
- **Verification:** `npx tsc -b` exits 0; `npx vitest run` 57/57 tests pass
- **Committed in:** `57c1b93` (Task 3 commit)

These success criteria require a real browser with an installed service worker and cannot be automated:

npx vite build && npx vite preview

2. DevTools → Application → Service Workers — confirm SW is active

4. Navigate to http://localhost:4173 — app shell must render from SW cache

**SC5 — PWA-05 (offline create):**
After SC4c is verified (SW installed):
1. Remain in offline mode (DevTools → Network → Offline)
2. Create a new entry via the app UI
3. DevTools → Application → IndexedDB → LifeLogDB → entries — entry must appear
4. Entry persists because IndexedDB writes require no network once the shell is cached

**Why automated is insufficient:** `virtual:pwa-register/react` and SW installation require the browser's Service Worker API — jsdom cannot simulate SW registration, cache population, or offline intercept. Build artifact assertions (`dist/sw.js`, `dist/manifest.webmanifest`) are the automated proxy.
