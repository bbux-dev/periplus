<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored sourceUrl → <a href> | An untrusted stored URL could carry a `javascript:` scheme; must not become a clickable href |
| stored metadata bag → DOM | Opaque extraction data rendered to the page; must be escaped |
| route :id → entry lookup | Arbitrary id string must resolve to a graceful not-found, never a crash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering/Spoofing | sourceUrl link render | mitigate | `isSafeUrl(entry.sourceUrl)` gates the `<a href>`; an unsafe scheme (e.g. `javascript:`) renders as plain text `<span>`, never an href. `rel="noopener noreferrer"` on safe links |
| T-06-02 | Tampering | metadata JSON preview | mitigate | Rendered via `JSON.stringify` inside a `<pre>` text node — React auto-escapes; no `dangerouslySetInnerHTML` |
| T-06-04 | Denial of Service | unknown :id | mitigate | id coerced to '' when missing; useEntry → null → graceful not-found view with Back |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies this phase |
</threat_model>
