<objective>
Create the navigation config module — the single source of truth for the Life Log nav tree.
A `NAVIGATION: DomainConfig[]` constant maps each `EntryDomain` to its display label, heroicon,
and ordered `EntryType[]`, plus a `getDomainConfig(domain)` lookup helper. Every Phase 3 screen
(and future capture phases) reads from this module instead of hardcoding the taxonomy.

Purpose: Keeps the dashboard, domain screens, and route table in sync with the
`EntryDomain`/`EntryType` taxonomy defined in `src/services/db.ts`, and resolves the `'expense'`
domain ambiguity (it belongs to both trips and expenditures) at the domain level.
Output: `src/config/navigation.ts` + `src/config/navigation.test.ts`.
</objective>
