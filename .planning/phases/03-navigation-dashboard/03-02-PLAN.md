```typescript
export interface EntryTypeConfig { type: EntryType; label: string; icon: HeroIcon }
export interface DomainConfig { domain: EntryDomain; label: string; icon: HeroIcon; types: EntryTypeConfig[] }
export const NAVIGATION: DomainConfig[]                       // [media, trips, expenditures]

```

Mobile-first layout convention (from WelcomePage.tsx): outer
`min-h-screen flex flex-col px-6 py-8 bg-[var(--color-background)] text-[var(--color-foreground)]`,
inner container `w-full max-w-sm mx-auto`. Theme tokens are CSS custom properties
(--color-primary/-foreground/-background/-muted/-border) defined in src/index.css.

RTL gotcha (react-router #12368): a component calling useParams/useNavigate MUST be rendered
inside `<MemoryRouter><Routes><Route .../></Routes></MemoryRouter>` — bare `<MemoryRouter>` is
insufficient. DashboardPage (no hooks beyond <Link>) may use the simpler wrapper; DomainPage
(uses useParams + useNavigate) MUST use the full Routes wrapper.

| route param → component | `:domain` is an untrusted string (a user can type any path). DomainPage reads it via `useParams` and looks it up. This is the one real robustness surface in this plan. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Denial of Service (crash) | DomainPage `:domain` lookup | mitigate | `getDomainConfig(domain)` returns `undefined` for unknown values; DomainPage renders a graceful "Unknown domain" branch instead of dereferencing undefined and crashing. Covered by the unknown-domain test. |
| T-03-03 | Tampering / Injection | DomainPage rendering of `domain` string | mitigate | The raw param is rendered as React text content only (auto-escaped), never as HTML, never passed to eval/dangerouslySetInnerHTML or a DB query. No injection surface. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages in Phase 3 (RESEARCH § Package Legitimacy Audit). No install task in this plan. |
</threat_model>
