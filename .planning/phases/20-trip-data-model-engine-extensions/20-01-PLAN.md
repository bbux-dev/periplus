<objective>
Extend the preserved Life Log engine's type layer and stamping path so it understands trips and
activities, WITHOUT a Dexie schema version bump. Three additive changes: (1) `EntryType` gains
`'trip'` and `'activity'`; (2) `ActiveMode` gains an optional `tripId` and `activateMode()` accepts
it as an optional 3rd parameter; (3) `draftToEntry` stamps `metadata.tripId` when the active mode
carries one.

Purpose: Every later v0.5.0 UI phase (21–24) depends on these engine primitives existing. This plan
is purely additive — no files are deleted, no behavior changes for non-trip entries.
Output: Updated `db.ts`, `entryFields.ts`, `activeMode.ts`, `captureService.ts` (+ their tests).
</objective>
